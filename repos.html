List of projects

Here follows a list of research software projects developed within the Heidelberg Collaboratory for Image Processing (HCI).

Click on the icon for more information on the individual projects. Some of these projects are hosted on the GitLab instance of the HCI: their home pages can be reached by clicking either on the project name or on the icon, and the latest version of the code can always be downloaded directly by clicking on the icon. All HCI-hosted projects are collectively grouped under the GitLab research_code group. You will need an HCI LDAP account in order to access these projects, please contact Dominic Spangenberger.

External project pages (e.g., on GitHub or BitBucket) are reachable via the icon. Bibliographical references are listed when available.

Project name Description Author(s) Bibrefs Links
CGP Cartesian grid partitioning. Thorben Kroeger, Niko Krasowski
This library depends on vecvec and chatty.
ccc3d A multicut solver based on CGP. Thorben Kroeger, Niko Krasowski
This library depends on CGP.
cgp13 Cartesian Grid Partitioning. Bjoern Andres, Thorben Kroeger
A C++ library for cartesian grid partitioning, with a matlab interface.
chatty A logger for both C++ and Python. Thorben Kroeger
This is a dependency for CGP.
cogstate Multitask learning using regularized multiple kernel learning. Melih Kandemir [0]
Code from the paper Multitask Learning Using Regularized Multiple Kernel Learning.
conservation_tracking_pipeline C++ Pipeline to run Conservation Tracking. Philipp Hanslovsky, Carsten Haubold, Martin Schiegg, David Stoeckel [1]
This pipeline runs the conservation tracking algorithm given a sequence of tiff images and pre-trained Random Forest classifiers. The classifiers can be extracted from previously trained ilastik projects for pixel classification and tracking, but every step will be rerun in the C++ pipeline (allows for batch-segmenting and tracking).
dpct Dynamic programming cell tracking. Carsten Haubold [2]
This is a stand-alone tool for running tracking of divisible objects using a modified successive shortest paths solver.
eventdetector Cell event detection. Melih Kandemir [3]
Code from the paper Cell event detection in phase-contrast microscopy sequences from few annotations.
fastfilters Fast gaussian and derivative convolutional filters. Sven Peter
Optimised for AVX.
histopath Histopathology cancer diagnosis by cell graphs. Melih Kandemir [4]
Code from the paper Empowering Multiple Instance Histopathology Cancer Diagnosis by Cell Graphs.
hytra Tracking Toolbox of the IAL Lab @ uni-heidelberg.de. Carsten Haubold, Steffen Wolf, Letitia Parcalabescu, Bernhard Kausler, Martin Schiegg
Tracking Toolbox of the IAL Lab @ uni-heidelberg.de.
ilastik Interactive image classification, segmentation and analysis. Christoph Sommer, Christoph Straehle, Ullrich Koethe, Fred A. Hamprecht and others [5]
ilastik is built as a modular software framework, which currently has workflows for automated (supervised) pixel- and object-level classification, automated and semi-automated object tracking, semi-automated segmentation and object counting without detection.
imgret Utilities for Gaussian processes for ML. Melih Kandemir
Utilities for Gaussian processes for ML.
libsvm_om Octave/Matlab interface for libsvm. Jun-Cheng Chen
This tool provides a simple interface to LIBSVM, a library for support vector machines (http://www.csie.ntu.edu.tw/~cjlin/libsvm).
miGraphPy A Python implementation of the multiple instance learning (MIL) method miGraph. Manuel Tuschen [6]
A Python implementation of the multiple instance learning (MIL) method miGraph.
mil_methods A collection of implementations of methods for multiple instance learning. Melih Kandemir
A collection of implementations of methods for multiple instance learning.
multiHypothesesTracking Tool for tracking of divisible objects. Carsten Haubold
This is a standalone tool for running tracking of divisible objects, with competing detection hypotheses in each frame. When specifying a ground truth labeling for a dataset, the weights can be learned using structured learning (by OpenGM's implementation of SBMRM). The tracking problem is then solved as ILP by Gurobi or CPLEX, depending on how this tool was compiled.
multitaskmkl Multitask learning using regularized multiple kernel learning. Melih Kandemir [7]
Code from the paper Multitask learning using regularized multiple kernel learning.
nifty A library for Multicuts and general image and graph segmentation. Thorsten Beier [8], [9]
A nifty library for Multicuts and general image and graph segmentation.
nnet_core Basic utilities for the implementation of neural networks in Python. Gregor Urban
Basic utilities for the implementation of neural networks in Python.
opengm OpenGM is a C++ template library for discrete factor graph models. Bjoern Andres, Thorsten Beier, Joerg H. Kappes
OpenGM includes state-of-the-art optimization and inference algorithms beyond message passing. OpenGM handles large models efficiently, since (i) functions that occur repeatedly need to be stored only once and (ii) when functions require different parametric or non-parametric encodings, multiple encodings can be used alongside each other, in the same model, using included and custom C++ code.
retinopathy Detection of retinopathy of prematurity using multiple instance learning. Melih Kandemir [10]
Code from the paper Detection of retinopathy of prematurity using multiple instance learning.
sstd_python Python / cython implementation of sparse space-time deconvolution for Calcium image analysis. Sven Peter [11]
Code from the paper Sparse space-time deconvolution for Calcium image analysis.
statistics-cgp Statistics for CGP. Thorben Kroeger
Evaluate average sizes of 1-sets, 2-sets, 3-sets as well as compression of topological grid blocks using the CGP library. This library depends on CGP.
struct-svm Structured SVM playground. Carsten Haubold
This is a python playground using dlib, gurobipy and opengm to explore structured svms and their solvers.
vecvec A serializable replacement for std::vector<std::vector<T>>. Thorben Kroeger
This is a dependency for CGP. Use case: loading a fixed inverse relabeling (one to many and many to many relation) of millions of objects from disk.
vigra Generic programming for computer vision. Ullrich Koethe and others
VIGRA stands for "Vision with Generic Algorithms". It's an image processing and analysis library that puts its main emphasis on customizable algorithms and data structures. VIGRA is especially strong for multi-dimensional images, because many algorithms (e.g. filters, feature computation, superpixels) are implemented for arbitrary high dimensions. By using template techniques similar to those in the C++ Standard Template Library, you can easily adapt any VIGRA component to the needs of your application, without thereby giving up execution speed. As of version 1.7.1, VIGRA also provides extensive Python bindings on the basis of the popular numpy framework.
visualizatino 2D and 3D visualisation of neurological datasets. Thorben Kroeger, Niko Krasowski
2D and 3D visualisation of neurological datasets.
